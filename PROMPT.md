## 4. 숫자패드 키 매핑

### 4-1. 점자 6점 매핑

```
숫자패드 7 → 점1 (idx0)   숫자패드 8 → 점4 (idx3)
숫자패드 4 → 점2 (idx1)   숫자패드 5 → 점5 (idx4)
숫자패드 1 → 점3 (idx2)   숫자패드 2 → 점6 (idx5)
```

배열 표현: `[dot1, dot2, dot3, dot4, dot5, dot6]` = `[idx0, idx1, idx2, idx3, idx4, idx5]`

### 4-2. 특수키

| 키 | 기능 |
|----|------|
| Numpad 0 | 띄어쓰기 |
| Numpad Enter | 전체 문장 읽기 |
| Numpad . (Decimal) | 한 글자 지우기 (단계적: 종성→중성→초성→확정텍스트) |

## 6. 핵심 구현 로직

### 6-1. useBrailleChording (Key Chording 엔진)

- `keydown` 시 눌린 키를 `Set`에 저장, `pressedDotsRef` 배열에 점 활성화
- `event.repeat` 철저히 무시
- 마지막 `keyup` 후 **50ms 디바운스** 타이머로 조합 확정 (`fireDots`)
- 모든 점자 키가 떨어진 시점에 `onChord` 콜백 호출

### 6-2. brailleConverter (한글 조합 상태 머신)

**HangulState 구조:**
```typescript
interface HangulState {
  cho: string    // 초성
  jung: string   // 중성
  jong: string   // 종성 (단일 or 겹받침 완성형)
  jong2: string  // 겹받침 두 번째 자음 (분리 대기)
}
```

**문맥 판단 (getNextContext):**
- `cho` 없음 → `chosung`
- `jung` 없음 → `jungsung`
- `jong2` 있음 → `chosung` (겹받침 완성, 다음은 새 음절)
- `jong` 없음 → `jongsung`
- 그 외 → `chosung`

**겹받침 처리:**
- `jong` + 새 자음 → `DOUBLE_JONGSUNG` 테이블 조회
- 성립 시: `jong` = 겹받침 완성형, `jong2` = 두 번째 자음 단독 보관
- 다음에 중성 입력 시: `jong` → `DOUBLE_JONGSUNG_FIRST`로 첫 자음만 받침 확정, `jong2` → 새 음절 초성

**쌍자음 처리:**
- 초성만 있고 중성 없는 상태에서 자음 입력 시 `SSANG_MAP` 조회
- 성립 시 초성을 쌍자음으로 교체

### 6-3. TTS (Web Speech API)

```typescript
lang = 'ko-KR'
pitch = 1.2
rate = 1.0
```

- 글자/자모 완성 시 즉시 읽기
- Numpad Enter: `"현재까지 입력한 문장은 [문장] 입니다."` (rate 0.9)
- Numpad .: `"지워졌어요"` 후 삭제
- 5초 무반응: `"무엇을 써볼까요? 숫자패드 7, 8, 4, 5, 1, 2번 키를 눌러보세요."`

### 6-4. 이중 입력 버그 방지

- `committedText`와 `hangulState`를 단일 `InputState` 객체로 통합
- `setInput()` 하나로만 업데이트 → React updater 함수 중첩 호출 제거
- `StrictMode` 제거 (`main.tsx`)
